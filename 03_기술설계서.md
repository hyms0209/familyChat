### 2.2 핵심 설계 원칙
1. **서버리스 우선** — MVP는 Firebase 위주. 확장기 비용/제어 필요 시 자체 서버 검토.
2. **클라이언트 직결 + Cloud Functions 보조** — 단순 CRUD는 Firestore 직접, 보안 민감/조합 로직은 Functions.
3. **가족 단위 데이터 격리** — Security Rules의 1차 방어선. 모든 컬렉션의 access는 `familyId` 일치 확인.
4. **오프라인 우선 UX** — Firestore 캐시 활용, 메시지 낙관적 업데이트.
5. **확장 가능한 모놀리식** — 마이크로서비스화 전 단일 backend (Functions monorepo)로 시작.

### 3.2 상태 관리 패턴

- **서버 상태**: TanStack React Query v5 — Firestore 실시간 구독을 query 로 래핑, 외부 REST API도 동일 패턴
- **클라이언트/UI 상태**: Zustand — `src/store/`에 도메인별 분리 (authStore, familyStore, locationStore, themeStore, notificationStore, devToolsStore)
- **폼 상태**: React Hook Form + Zod 검증
- **글로벌 컨텍스트**: Provider (FamilyProvider, AuthCheckProvider 등) — `useFamily()`, `useAuth()` 훅으로 접근
- **폼별 로컬 상태**: 화면별 `use{X}Screen.ts` 훅에서 useState/useReducer

### 3.3 내비게이션 (poc-bank Router 패턴)

#### 3.3.1 핵심 컴포넌트

- **`src/navigation/Router.ts`**: 전역 `navigationRef` + 타입 안전 메서드
  - `Router.moveToMenu(MENU.X.Y, params)` — 전진
  - `Router.replaceToMenu(MENU.X.Y, params)` — 현재 스크린 교체
  - `Router.pop({ screenType, data })` — 결과 반환과 함께 pop
  - `Router.popToMenu(MENU.X.Y, params)` — 특정 화면까지 pop
  - `Router.moveToMain()` — 메인으로 스택 리셋
  - `Router.moveToTabMenu(MENU.MAIN.HOME)` — 탭 전환
  - `navigationRef` 준비 대기 로직 내장 (재시도 포함)

```tsx
// src/providers/ComposeProviders.tsx
type ProviderComp = React.ComponentType<{ children: React.ReactNode }>

export function composeProviders(...providers: ProviderComp[]) {
    return ({ children }: { children: React.ReactNode }) =>
        providers.reduceRight(
            (acc, Provider) => <Provider>{acc}</Provider>,
            children as React.ReactElement,
        )
}
```

### 5.2 Firestore Security Rules (핵심 발췌)

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    function isAuth() { return request.auth != null; }
    function uid() { return request.auth.uid; }
    function isFamilyMember(familyId) {
      return isAuth()
        && familyId in request.auth.token.familyIds;
    }
    function isParent(familyId) {
      return isFamilyMember(familyId)
        && (request.auth.token.role == "parent"
            || request.auth.token.role == "owner");
    }
    function isOwner(familyId) {
      return isFamilyMember(familyId)
        && request.auth.token.role == "owner";
    }

    match /users/{userId} {
      allow read: if isAuth() && uid() == userId;
      allow write: if isAuth() && uid() == userId;
    }

    match /families/{familyId} {
      allow read: if isFamilyMember(familyId);
      allow update: if isParent(familyId)
        && request.resource.data.ownerId == resource.data.ownerId;
      allow delete: if isOwner(familyId);

      match /members/{userId} {
        allow read: if isFamilyMember(familyId);
        allow create, delete: if isParent(familyId);
        allow update: if isParent(familyId) || uid() == userId;
      }

      match /chats/{chatId} {
        allow read: if isFamilyMember(familyId)
          && uid() in resource.data.participantIds;
        allow create: if isFamilyMember(familyId)
          && uid() in request.resource.data.participantIds;
      }

      match /chats/{chatId}/messages/{messageId} {
        allow read: if isFamilyMember(familyId)
          && uid() in get(/databases/$(database)/documents/families/$(familyId)/chats/$(chatId)).data.participantIds;
        allow create: if isFamilyMember(familyId)
          && request.resource.data.senderId == uid();
        allow update, delete: if isFamilyMember(familyId)
          && resource.data.senderId == uid()
          && request.time < resource.data.sentAt + duration.value(5, 'm');
      }

      match /events/{eventId} {
        allow read: if isFamilyMember(familyId);
        allow create: if isFamilyMember(familyId)
          && request.resource.data.createdBy == uid();
        allow update, delete: if isParent(familyId)
          || (isFamilyMember(familyId) && resource.data.createdBy == uid());
      }

      match /academies/{academyId} {
        allow read: if isFamilyMember(familyId);
        allow write: if isParent(familyId);
      }

      match /locations/{userId} {
        allow read: if isFamilyMember(familyId);
        allow write: if isFamilyMember(familyId) && uid() == userId;
      }

      match /invites/{inviteId} {
        allow read: if isFamilyMember(familyId);
        allow create, update: if isParent(familyId);
      }
    }

    match /locationHistory/{familyId}/users/{userId}/points/{pointId} {
      allow read: if isFamilyMember(familyId);
      allow write: if isFamilyMember(familyId) && uid() == userId;
    }

    match /notifications/{userId}/items/{notifId} {
      allow read, update: if isAuth() && uid() == userId;
      allow create: if false; // Cloud Functions만 생성
    }
  }
}
```

```typescript
// functions/src/family/consumeInvite.ts
import { onCall, HttpsError } from "firebase-functions/v2/https";
import { admin } from "../shared/admin";

export const consumeInvite = onCall(
  { region: "asia-northeast3", enforceAppCheck: true },
  async (request) => {
    const { inviteId } = request.data;
    const uid = request.auth?.uid;
    if (!uid) throw new HttpsError("unauthenticated", "");

    const db = admin.firestore();
    const inviteRef = db.collectionGroup("invites").where("inviteId", "==", inviteId);
    const snap = await inviteRef.get();
    if (snap.empty) throw new HttpsError("not-found", "초대를 찾을 수 없어요");
    const invite = snap.docs[0];

    const data = invite.data();
    if (data.revoked || data.consumedBy) {
      throw new HttpsError("failed-precondition", "사용된 초대입니다");
    }
    if (data.expiresAt.toMillis() < Date.now()) {
      throw new HttpsError("failed-precondition", "만료된 초대입니다");
    }

    const familyId = invite.ref.parent.parent!.id;
    await db.runTransaction(async (tx) => {
      tx.set(
        db.doc(`families/${familyId}/members/${uid}`),
        {
          userId: uid,
          role: "child",          // 기본값. 가입 후 부모가 변경 가능
          joinedAt: admin.firestore.FieldValue.serverTimestamp(),
          active: true,
        }
      );
      tx.update(invite.ref, { consumedBy: uid });
    });

    // Custom Claim 업데이트는 setFamilyClaims 트리거가 처리
    return { familyId };
  }
);
```

## 7. 핵심 기능 구현 상세

### 7.1 채팅 (Realtime)

**전송 흐름**
1. 클라이언트: 메시지 객체 생성 (status="sending")
2. Firestore에 add → onSnapshot으로 즉시 본인 화면 갱신 (낙관적)
3. `onMessageCreate` Functions:
   - 부모 chat doc의 `lastMessage` 갱신
   - 다른 참여자에게 FCM 발송
4. 클라이언트: snapshot 갱신으로 status="sent" → 모두 읽음 시 "read"

**오프라인 처리**
- Firestore SDK 오프라인 캐시 활성화
- 송신 실패 메시지는 로컬 큐 (AsyncStorage) → 재연결 시 자동 재시도
- 사용자에게 "전송 대기 중" 표시

### 7.2 실시간 위치 공유

**클라이언트 (RN)**
- 라이브러리: `expo-task-manager` + `expo-location`
- 백그라운드 태스크 등록:
  ```ts
  TaskManager.defineTask(LOCATION_TASK, async ({ data }) => {
    const { locations } = data;
    const last = locations[locations.length - 1];
    await db.doc(`families/${familyId}/locations/${uid}`).set({
      userId: uid,
      lat: last.coords.latitude,
      lng: last.coords.longitude,
      accuracy: last.coords.accuracy,
      updatedAt: serverTimestamp(),
      source: "background",
    });
  });
  ```
- `Location.startLocationUpdatesAsync(LOCATION_TASK, { accuracy, deferredUpdatesDistance, ... })`

**적응적 갱신**
- 정지 감지(가속도/속도)로 갱신 주기 5분으로 늘림
- 배터리 < 20% 시 자동 5분
- iOS Significant Location Change 활용

**히스토리 저장**
- `onLocationUpdate` Functions가 `locationHistory`에 점 추가
- TTL은 Firebase Firestore TTL 정책 활용 (free=24h, premium=30d/1y)

**프라이버시 가드**
- 로그에 위치 좌표 절대 기록 금지
- 클라이언트 캐시 위치도 24h 후 자동 폐기

### 7.3 캘린더 / 학원 일정 통합

**학원 → 캘린더 표시 전략 (선택)**

옵션 A — **events에 미리 펼쳐 저장 (Materialized)**
- `onAcademyCreate`/`onAcademyUpdate`가 6개월치 events를 미리 생성
- 장점: 쿼리 단순, Geofence·알림 처리 간편
- 단점: 학원 변경 시 대량 갱신, 저장 비용

옵션 B — **events에 RRULE만 저장 (Lazy)**
- 학원당 1개 event with rrule
- 클라이언트가 화면 표시 시 펼침
- 장점: 저장 효율, 변경 즉시 반영
- 단점: 알림 스케줄러가 RRULE 풀이 필요

**채택: 하이브리드**
- 마스터: `academies` 컬렉션 (RRULE 정의)
- 표시: 클라이언트가 `academies` + 일반 `events`를 머지하여 캘린더 그림
- 알림: `scheduleEventReminders` cron이 24시간 윈도우 내 학원 이벤트를 펼쳐 알림 큐에 적재

### 7.4 푸시 알림 (FCM)

**토큰 관리**
- 앱 시작 시 `messaging().getToken()` → `users/{uid}/devices/{deviceId}` 저장
- 로그아웃 시 토큰 삭제

**페이로드 표준**
```json
{
  "notification": { "title": "...", "body": "..." },
  "data": {
    "type": "chat" | "event" | "location" | "academy" | "system",
    "deeplink": "familyhub://chat/{chatId}",
    "groupKey": "family-{familyId}-chat",
    "silent": "false"
  },
  "apns": { "payload": { "aps": { "thread-id": "..." } } },
  "android": { "notification": { "channel_id": "chat" } }
}
```

**알림 그룹화**
- iOS: thread-id로 묶음
- Android: channel + group으로 분리

### 7.5 가족 초대 (Deeplink + QR)

- 토큰 생성: `crypto.randomBytes(24).toString('base64url')` (32자)
- 링크: `https://familyhub.app/invite/{token}` (universal link / app link)
- QR: 같은 URL을 QR 인코딩
- `consumeInvite` Function이 검증 + 가족 가입 처리

## 12. 확장성 고려

### 12.1 1만 가족 도달 시

- 채팅 트래픽이 많은 가족은 별도 채팅 인프라 검토 (Stream, Sendbird 또는 자체 WebSocket).
- 위치 처리는 Cloud Run + Pub/Sub로 이전 검토.

## 13. 데이터 다운로드 / 삭제 (PIPA)

### 13.1 데이터 다운로드 흐름
1. 사용자가 [설정 → 개인정보 → 데이터 다운로드] 요청

2. 30일 유예 기간 안내 (취소 가능)
3. 만료 시 모든 컬렉션에서 사용자 데이터 삭제, 채팅 메시지는 "(탈퇴한 사용자)" 처리.
4. 삭제 완료 알림 메일 발송.
