## 3. 프롬프트 표준 포맷

모든 프롬프트는 다음 구조를 따른다:

```markdown
### P-{단계}.{번호}. {제목}

**[목적]**
이 프롬프트로 무엇을 얻고자 하는지 한 줄.

**[참조 문서]**
- `04_와이어프레임.md` §X.Y
- (필요 시) `02_기능명세서.md` §F##

**[전제 조건]**
- 이전 단계에서 완료되어 있어야 할 것
- 열려 있어야 할 .pen 파일

**[프롬프트]** (← 이 블록을 복사해서 사용)
```
실제로 Claude/Pencil에 입력할 텍스트
```

**[제약 조건]**
- 토큰 사용 규칙
- 프레임 크기
- 한국어/영어 표기 규칙

**[성공 기준]**
- [ ] 결과물 체크리스트 1
- [ ] 결과물 체크리스트 2

**[다음 단계]**
→ P-{다음 번호}
```

## 4. 사용 팁

### 4.1 Pencil에서 실행하기
1. Pencil 캔버스 열기
2. `Cmd/Ctrl + K` 누르기
3. **[프롬프트]** 블록의 텍스트만 복사 → 붙여넣기 → 실행
4. 결과 확인 후 **[성공 기준]** 체크리스트 검증

### 4.2 Claude Code 채팅창에서 실행하기 (코드 변환용)
1. Cursor의 Claude Code 패널 열기
2. **[프롬프트]** 텍스트 붙여넣기
3. 코드 산출물은 자동으로 해당 위치에 저장됨 (`src/...`)

### 4.3 프롬프트 변형
- 각 프롬프트의 `{변수}` 부분 (예: `{화면 이름}`, `{컴포넌트 이름}`)을 본인 상황에 맞게 치환
- 프롬프트 끝에 "그리고 ○○도 추가해줘" 식으로 확장 가능

### 4.4 결과 일관성 유지
- 한 세션에서 가급적 시퀀스대로 실행 (앞 결과를 다음 프롬프트가 참조)
- 새 세션 시작 시 첫 프롬프트로 "이전에 만든 디자인 시스템과 컴포넌트를 참조해서 진행해줘" 한 줄 추가하면 일관성↑

## 5. 트러블슈팅

| 상황 | 해결 |
|---|---|
| 결과가 와이어프레임 명세와 다르게 나옴 | 프롬프트에 "정확히 04_와이어프레임.md §X.Y의 텍스트와 컴포넌트 명세를 따라줘"를 추가 |
| 컬러가 토큰 대신 임의 hex로 나옴 | "07_프롬프트_디자인시스템.md에서 정의한 토큰 변수만 사용해줘"로 강제 |
| 한국어가 깨짐 | Pencil 폰트 설정 확인 (Pretendard 또는 Apple SD Gothic Neo 추가) |
| 너무 큰 결과물로 캔버스 어수선 | "한 캔버스에 가로 4 × 세로 5 그리드로 정렬해줘"를 추가 |
| 코드가 NativeWind 대신 StyleSheet로 나옴 | "NativeWind 클래스만 사용. StyleSheet 금지"를 명시 |
