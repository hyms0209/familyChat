## P-1.1. 디자인 시스템 토큰 라이브러리 생성

**[목적]**
`04_와이어프레임.md` §1에 정의된 디자인 토큰을 시각적 라이브러리로 만들어, 이후 모든 컴포넌트·화면이 이 토큰만 참조하도록 한다.

**[참조 문서]**
- `04_와이어프레임.md` §1.1 ~ §1.6 (디바이스, 컬러, 타이포, 스페이싱, 모서리, 그림자)

**[전제 조건]**
- Pencil 캔버스 열기 → 새 .pen 파일 생성: `design/01_design_system.pen`
- 빈 캔버스 상태

**[프롬프트]**
```
지금 열려 있는 .pen 캔버스에 가족 커뮤니케이션 앱의 "디자인 시스템 라이브러리"를 시각화해줘.

아래 워크스페이스 파일을 참조해 정확히 따라줘:
/Users/lmh/Documents/Claude/Projects/가족채팅/04_와이어프레임.md 의 §1 전체

캔버스에 다음 5개 섹션을 가로로 배치 (각 섹션 사이 64px 간격):

1) 컬러 팔레트
   - 라이트 모드 12개 토큰: primary, primary-soft, secondary, bg, surface, text-primary, text-secondary, divider, success, warning, error, overlay
   - 각 컬러는 80×80 스와치 + 토큰명 + hex 값
   - 다크 모드 12개 토큰도 같은 형태로 그 아래에
   - 가족 컬러 8개(#FF7E6B 등)도 별도 줄에

2) 타이포그래피 9단계
   - display(32) / heading-1(24) / heading-2(20) / heading-3(18) / body(16) / body-bold(16) / caption(13) / caption-bold(13) / tiny(11)
   - 각 샘플 텍스트는 "안녕, 우리 가족" + 영어 "Hello Family"
   - 좌측에 토큰명, 우측에 size/weight/line-height

3) 스페이싱 토큰
   - xs(4), sm(8), md(12), lg(16), xl(24), 2xl(32), 3xl(48)
   - 각 값을 회색 막대 길이로 시각화

4) 모서리 반경
   - button(12), card(16), modal(20), chip(full), avatar(full)
   - 각 라벨이 적힌 회색 박스 5개

5) 그림자
   - sm/md/lg 3개 카드 샘플, 흰 배경, 모서리 16

전체 캔버스 크기: 1440×1024 정도. iPhone 14 프레임 안에 들어갈 필요 없음 (디자인 시스템은 데스크톱 캔버스로).

컬러 토큰명은 영어 그대로 (--color-primary 등), 한국어 라벨은 토큰명 우측에 작게.
```

## P-1.2. 다크 모드 토큰 검증 + 가족 컬러 정렬

**[목적]**
P-1.1에서 만든 토큰 라이브러리에서 다크 모드 토큰의 가독성과 가족 컬러의 시각적 균형을 확인하고 보정한다.

**[참조 문서]**
- `04_와이어프레임.md` §1.2 (컬러), §7.3 (색상 대비 WCAG AA)

**[전제 조건]**
- P-1.1 완료, `01_design_system.pen` 캔버스가 열려 있음

**[프롬프트]**
```
방금 만든 디자인 시스템 캔버스에서 다음 검증 작업을 해줘:

1) 다크 모드 컬러 12개의 가독성 점검
   - 각 다크 토큰 위에 본문 텍스트("가족과 함께")를 올렸을 때 WCAG AA(4.5:1)를 만족하는지 확인
   - 만족 못 하는 경우 우측에 ⚠ 표시 + 권장값 제안

2) 가족 컬러 8개의 시각적 균형 점검
   - 8개 컬러가 명도/채도가 너무 튀거나 비슷한 게 있는지 확인
   - 각 컬러 위에 흰 텍스트와 검정 텍스트를 모두 올려, 어느 쪽이 더 적합한지 ✓ 표시
   - 5~8번(Premium)은 약간 더 채도를 낮춰 무료 컬러 4개와 차별

3) 라이트/다크 컬러 비교
   - 같은 토큰의 라이트/다크 두 값을 나란히 놓고 명도 차이가 일관되는지

4) 결과를 캔버스 우측에 별도 "검증 결과" 섹션으로 추가
   - 통과: ✓ 초록
   - 경고: ⚠ 주황 + 메모
   - 실패: ✗ 빨강 + 권장 hex

04_와이어프레임.md §7.3 접근성 노트도 같이 반영해줘.
```

**[제약 조건]**
- 기존 토큰을 즉시 변경하지 말 것 (제안만)
- 변경 제안은 추후 사용자 승인 후 적용

**[성공 기준]**
- [ ] 다크 토큰 12개 모두 대비 점수 표시
- [ ] 가족 컬러 8개 위 텍스트 컬러 권장 표시
- [ ] WCAG AA 미달 토큰에 ⚠ + 대안 hex 제안
- [ ] 검증 결과 섹션이 캔버스 우측에 명확히 분리
