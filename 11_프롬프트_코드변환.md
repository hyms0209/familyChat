위 부트스트랩 프롬프트는 작업이 매우 커서 Cursor의 Claude Code 한 응답으로 처리하기 어려울 수 있다 (도구 호출 한계 또는 토큰 한도). 한 번 실행해서 5분 이상 진전이 없거나 freeze되면 **4단계로 쪼갠 분할 버전**을 사용하라.

각 분할 파일은 한 응답 안에 안전하게 들어갈 크기로 설계되어 있다.

| 단계 | 파일 | 시간 | 산출물 |
|:---:|---|:---:|---|
| A | `13_부트스트랩_A_분석.md` | 1~3분 | poc-bank 매핑 표 (파일 생성 X) |
| B | `13_부트스트랩_B_프로젝트.md` | 5~10분 | mobile/ Expo 프로젝트 + 의존성 |
| C | `13_부트스트랩_C_핵심이식.md` | 5~10분 | poc-bank Tailwind/Router/Provider/Pretendard 이식 |
| D | `13_부트스트랩_D_도메인.md` | 5~10분 | 가족채팅 MENU/Provider/screens 스캐폴드 + Git |

**사용법**: Cursor의 Claude Code 채팅창에 한 줄씩.

```
@13_부트스트랩_A_분석.md 그대로 실행해줘.
```
완료 확인 후:
```
@13_부트스트랩_B_프로젝트.md 그대로 실행해줘.
```
이런 식으로 A → B → C → D 순서대로. 각 단계 완료 후 다음으로.

분할 버전을 사용하면 한 응답에 너무 많은 작업이 몰리지 않아 freeze 위험이 거의 없고, 단계별로 결과 검증이 가능하다.

```
다음 검증을 수행해줘:

1) 토큰 일관성:
   - mobile/src/ 안에 hardcoded hex 컬러(/#[0-9A-Fa-f]{6}/)가 토큰 정의 외에 있는지 grep
   - 발견되면 토큰으로 치환 제안

2) NativeWind만 사용:
   - StyleSheet.create 사용처 grep → 0건이어야 함

3) 컴포넌트 재사용:
   - 화면 파일에서 직접 View + 스타일을 만든 게 있다면 → 공통 컴포넌트로 치환 제안

4) 접근성:
   - 모든 Button/Input/Press 가능 요소에 accessibilityLabel 있는지

5) TypeScript:
   - npx tsc --noEmit 결과
   - any 타입 사용처 (가능하면 제거)

6) Firestore 쿼리:
   - 직접 firestore() 호출이 도메인 훅 외에 있는지

발견된 항목을 markdown 표로 보고: 파일 / 줄 / 문제 / 제안 수정.
```
