## 1. 정보 구조 (Information Architecture)

### 1.1 최상위 내비게이션 (하단 탭 5개)

```
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│   홈     │  채팅    │  지도    │ 캘린더   │  더보기  │
│  (Home)  │ (Chat)   │ (Map)    │(Calendar)│ (More)   │
└──────────┴──────────┴──────────┴──────────┴──────────┘
```

- **홈**: 오늘의 가족 요약 (현재 위치, 다음 일정, 새 메시지, 학원 알림)
- **채팅**: 그룹 채팅 + 1:1 메시지 목록
- **지도**: 가족 구성원 실시간 위치
- **캘린더**: 월/주/일 일정 + 학원 시간표
- **더보기**: 가족 관리, 프로필, 설정, 알림함

### 1.2 화면 맵 (요약)

```
[로그인/가입]
  ├─ 시작 화면 → 로그인 / 회원가입
  ├─ 인증 (이메일·전화·소셜)
  ├─ 프로필 설정 (닉네임, 사진, 역할)
  └─ 가족 만들기 / 가족 가입

[홈 탭]
  └─ 오늘 요약 카드 스택

[채팅 탭]
  ├─ 채팅 목록 (그룹 + 1:1)
  └─ 채팅방
      ├─ 메시지 작성 (텍스트, 이모지, 이미지)
      └─ 메뉴 (사진 보내기, 위치 공유)

[지도 탭]
  └─ 지도 + 가족 핀
      ├─ 가족 구성원 카드
      └─ 위치 공유 설정

[캘린더 탭]
  ├─ 월 뷰 / 주 뷰 / 일 뷰
  ├─ 일정 상세
  └─ 일정/학원 시간표 추가

[더보기 탭]
  ├─ 가족 관리 (구성원, 초대)
  ├─ 알림함
  ├─ 내 프로필
  └─ 설정 (알림, 위치, 개인정보, 약관, 로그아웃)
```

## 2. 사용자 권한 모델

### 2.1 역할 정의

| 역할 | 설명 | 권한 |
|---|---|---|
| **owner (가족 생성자)** | 가족을 만든 사람. 한 가족당 1명 (양도 가능) | 모든 권한 + 가족 삭제 |
| **parent (부모)** | 가족 관리 권한 | 구성원 초대/제거, 일정·학원 관리, 모든 위치 조회 |
| **child (자녀)** | 일반 구성원 | 채팅, 본인 위치 공유, 일정 조회/추가, 본인 일정만 수정 |
| **guardian (보호자, 확장)** | 조부모 등 부 보호자. v3에서 활성화 | parent와 유사하나 결제/탈퇴 권한 없음 |

### 2.2 권한 매트릭스 (MVP)

| 액션 | owner | parent | child |
|---|:---:|:---:|:---:|
| 가족 삭제 | ✅ | ❌ | ❌ |
| owner 양도 | ✅ | ❌ | ❌ |
| 구성원 초대 | ✅ | ✅ | ❌ |
| 구성원 강제 퇴장 | ✅ | ✅ | ❌ |
| 자기 자신 탈퇴 | ✅ (양도 후) | ✅ | ✅ |
| 그룹 채팅 메시지 | ✅ | ✅ | ✅ |
| 1:1 메시지 | ✅ | ✅ | ✅ |
| 메시지 삭제 (본인) | ✅ | ✅ | ✅ |
| 메시지 삭제 (타인) | ✅ | ✅ (자녀 발신만) | ❌ |
| 자기 위치 공유 ON/OFF | ✅ | ✅ | ✅ |
| 가족 위치 조회 | ✅ | ✅ | ✅ |
| 일정 생성/수정/삭제 (전체) | ✅ | ✅ | ❌ |
| 본인 일정 생성/수정 | ✅ | ✅ | ✅ |
| 학원 시간표 등록/수정 | ✅ | ✅ | ❌ |

## 3. 기능 명세

### F1. 회원가입 / 로그인

#### F1.1 개요
이메일 / 전화번호 / 소셜 로그인(Apple, Google, Kakao) 중 선택. 만 14세 미만은 별도 흐름.

#### F1.2 User Stories
- **US-1.1** 사용자로서, 이메일과 비밀번호로 가입할 수 있다.
- **US-1.2** 사용자로서, Apple/Google/Kakao 계정으로 한 번에 로그인할 수 있다.
- **US-1.3** 만 14세 미만 자녀의 경우, 부모의 동의 후에만 가입할 수 있다.
- **US-1.4** 사용자로서, 비밀번호를 잃어버려도 이메일로 재설정할 수 있다.

#### F1.3 사용자 플로우

**일반 가입 (만 14세 이상)**
1. 시작 화면 → "가입하기" 탭
2. 인증 수단 선택 (이메일 / 전화 / 소셜)
3. 인증 정보 입력 + 인증 코드 검증
4. 약관 동의 (필수: 이용약관, 개인정보처리방침 / 선택: 마케팅)
5. 프로필 설정 (닉네임, 프로필 사진 선택, 역할 선택: 부모/자녀)
6. 가족 만들기 / 초대 코드로 가입

**자녀 가입 (만 14세 미만)**
1. "자녀 계정 만들기" 선택 시 부모 인증 화면
2. 부모 본인 인증 (휴대폰 본인인증 또는 OTP)
3. 법정대리인 동의 명시적 체크
4. 자녀 프로필 설정 (닉네임, 생년월일, 사진은 선택)
5. 자녀가 직접 입력한 비밀번호는 부모에게도 비공개. 단, 부모는 임시 비밀번호 발급 가능.

#### F1.4 상세 규칙
- 닉네임: 1~20자, 한글/영어/숫자/이모지 허용
- 비밀번호: 8자 이상, 영문+숫자 필수, 특수문자 권장
- 이메일: 가입 시 인증 메일 발송, 24시간 내 인증 안 하면 미가입으로 정리
- 전화번호: 한국 +82 / 010 등 검증
- 소셜 로그인 첫 사용 시 위 약관 동의·프로필 설정 단계 통과 필수

#### F1.5 빈 상태 / 에러
- 인증 코드 만료: "인증 코드가 만료됐어요. 다시 받기"
- 이미 가입된 이메일: "이미 가입된 계정이에요. 로그인하시겠어요?"
- 비밀번호 잘못 5회 이상: 5분 잠금

### F2. 가족 생성·초대·가입

#### F2.1 개요
가족 그룹을 생성하고, 링크/QR로 다른 가족 구성원을 초대한다. 한 사용자는 동시에 여러 가족에 속할 수 있다 (Phase 2 — MVP는 1가족 제한).

#### F2.2 User Stories
- **US-2.1** owner로서, 가족 그룹을 만들고 이름·사진을 설정할 수 있다.
- **US-2.2** owner/parent로서, 초대 링크 또는 QR을 가족에게 공유할 수 있다.
- **US-2.3** 신규 사용자로서, 받은 링크를 통해 가족에 가입할 수 있다.
- **US-2.4** parent로서, 자녀에게 직접 초대 코드를 입력시켜 가입시킬 수 있다.
- **US-2.5** owner로서, 가족 구성원을 강제 퇴장시킬 수 있다 (1단계 확인).

#### F2.3 사용자 플로우

**가족 만들기**
1. 가입 직후 또는 [더보기 → 가족 만들기]
2. 가족 이름 입력 (예: "임가족")
3. (선택) 가족 대표 사진 업로드
4. 본인 역할 확인 (parent/child)
5. 완료 → 가족 ID 발급, owner = 본인

**초대 (링크/QR)**
1. [더보기 → 가족 관리 → 초대하기]
2. 두 가지 옵션 중 선택:
   - **링크 공유**: 임시 토큰이 포함된 deeplink 생성. 카카오톡/문자/메일로 공유.
   - **QR 코드**: 같은 토큰의 QR 표시. 자녀 폰을 부모가 직접 스캔.
3. 토큰 만료: 기본 24시간, owner가 즉시 폐기 가능.
4. 초대 받은 사람이 링크 탭 → 앱 실행 → 자동으로 "임가족에 가입하시겠어요?" 다이얼로그
5. 동의 시 가족 가입 완료, 가족 채팅에 시스템 메시지 ("민서 님이 가족에 합류했어요").

**구성원 강제 퇴장**
1. [가족 관리 → 구성원 → 해당 인물 → 퇴장시키기]
2. 확인 다이얼로그 ("정말 퇴장시킬까요? 이전 메시지는 남지만 이후엔 접근할 수 없어요")
3. 즉시 처리, 시스템 메시지로 채팅에 안내

#### F2.4 상세 규칙
- 가족 이름: 1~20자
- 가족당 최대 구성원: MVP는 4명 (Premium에서 8명)
- 초대 링크 토큰: 무작위 32자, HTTPS deeplink
- 가족 가입 시 채팅에 자동 시스템 메시지
- 자녀(만 14세 미만)가 초대 받은 경우: 부모 가입 절차 우선 진행

#### F2.5 빈 상태 / 에러
- 만료된 링크: "초대 링크가 만료됐어요. 가족에게 새 링크를 요청하세요"
- 가족이 가득 참: "이 가족은 인원이 가득 찼어요. 프리미엄에서 8명까지 가능해요"
- 이미 가입된 가족: "이미 ○○가족에 가입되어 있어요"

### F3. 그룹 채팅

#### F3.1 개요
가족 전체가 참여하는 단일 그룹 채팅방. 가족 가입 시 자동 입장.

#### F3.2 User Stories
- **US-3.1** 구성원으로서, 텍스트/이모지/이미지 메시지를 보낼 수 있다.
- **US-3.2** 구성원으로서, 다른 구성원이 메시지를 읽었는지 알 수 있다.
- **US-3.3** 구성원으로서, 내가 보낸 메시지를 5분 내 삭제할 수 있다.
- **US-3.4** parent로서, 자녀의 부적절한 메시지를 가릴 수 있다 (숨김).
- **US-3.5** 구성원으로서, 위치 핀이나 일정 카드를 채팅에 첨부할 수 있다.

#### F3.3 사용자 플로우

**메시지 송수신**
1. [채팅 탭 → 가족 채팅방]
2. 입력창에 텍스트 작성 → 전송
3. 화면 즉시 갱신 (낙관적 업데이트)
4. 서버 도달 후 체크 표시(✓), 모든 가족 읽음 시 (✓✓)

**이미지 첨부**
1. 입력창 좌측 [+] → 사진 선택
2. 갤러리 또는 카메라 선택
3. 사진 미리보기 + 캡션 작성
4. 전송 → 업로드 진행 표시 → 완료

**메시지 삭제**
1. 본인 메시지 길게 누르기 → "삭제"
2. 5분 이내: 모두에게서 삭제 ("이 메시지는 삭제되었습니다")
3. 5분 이후: 본인 화면에서만 숨김

**시스템 메시지**
- 가족 합류/탈퇴, 일정 추가, 학원 변경 등 자동 안내
- 일반 메시지와 다른 톤(연한 회색 카드)

#### F3.4 상세 규칙
- 메시지 길이: 텍스트 최대 4,000자
- 이미지: 최대 10MB, 자동 압축(긴 변 1920px 가정), 가족 단위 스토리지 사용량 추적
- 메시지 보관: 영구 (삭제 시점까지)
- 읽음 표시: 가족 구성원별 마지막 읽은 메시지 시각 기록
- 입력 중 표시: "○○ 입력 중..."
- 알림: 가족 채팅의 새 메시지 → 가족 푸시 알림

#### F3.5 빈 상태 / 에러
- 새 가족 첫 채팅: 환영 시스템 메시지 1건만 표시
- 네트워크 오류: 메시지 옆 ⚠ 표시 + "다시 보내기" 액션
- 이미지 업로드 실패: 재시도 버튼

### F4. 1:1 다이렉트 메시지

#### F4.1 개요
가족 구성원 간 개인 대화. 자녀-부모 별도 대화, 부부 별도 대화 등에 사용.

#### F4.2 User Stories
- **US-4.1** 구성원으로서, 다른 구성원과 1:1 대화를 시작할 수 있다.
- **US-4.2** 1:1 대화는 가족 그룹 채팅과 분리된 별도 공간이다.
- **US-4.3** 1:1 대화 내용은 그룹 채팅에 노출되지 않는다.

#### F4.3 사용자 플로우
1. [채팅 탭] 우상단 [+] → "1:1 메시지"
2. 가족 구성원 목록에서 선택 (본인 제외)
3. 대화방 진입 (이전 대화가 있으면 이어서 표시)
4. F3와 동일한 메시지 송수신 UX

#### F4.4 상세 규칙
- 1:1 대화는 가족 ID 내에서만 가능 (외부와는 불가능)
- 그룹 채팅과 동일한 보관/삭제 규칙
- 자녀-부모 1:1 대화는 다른 부모도 별도 인증 없이 조회 불가 (사적 영역 보호)
- 단, 이용약관 위반 신고 시 운영팀이 메타데이터 조회 가능 (메시지 내용은 별도 절차 필요)

#### F4.5 빈 상태 / 에러
- 가족 구성원이 본인뿐인 경우: "1:1 대화할 가족이 아직 없어요. 가족을 초대해 보세요"

### F5. 실시간 위치 공유

#### F5.1 개요
가족 구성원 간 백그라운드 실시간 위치 공유. 외부에는 절대 노출되지 않으며, 본인이 언제든 일시 정지 가능.

#### F5.2 User Stories
- **US-5.1** 구성원으로서, 내 위치를 가족에게 실시간으로 공유할 수 있다.
- **US-5.2** 구성원으로서, 가족 구성원의 현재 위치를 지도에서 확인할 수 있다.
- **US-5.3** 구성원으로서, 위치 공유를 일시 정지(예: 1시간) 또는 영구 끄기 할 수 있다.
- **US-5.4** 자녀로서, 부모가 내 위치를 보는 것을 알고 있고, 일시 정지 권한이 있다.
- **US-5.5** 구성원으로서, 가족의 마지막 위치 갱신 시각을 알 수 있다.

#### F5.3 사용자 플로우

**최초 활성화**
1. 가입 후 첫 [지도 탭] 진입
2. 위치 공유 동의 화면:
   - "가족과 위치를 공유할까요?"
   - 약관: 위치정보 이용 동의 (필수, 미동의 시 위치 기능 사용 불가)
3. 시스템 위치 권한 요청 (iOS: '항상 허용' 권장, Android: 백그라운드 권한)
4. 백그라운드 권한 부여 가이드 (단계별 설명, 미부여 시 'Foreground only' 모드)
5. 첫 위치 갱신 → 지도에 본인 + 가족 핀 표시

**일시 정지 / 끄기**
1. [지도 탭 → 우상단 본인 카드 → 위치 공유]
2. 옵션:
   - 1시간 정지 / 4시간 정지 / 오늘 하루 정지
   - 영구 끄기 (가족에게 알림으로 안내됨)
3. 정지 중에는 마지막 위치만 가족에게 보임 + "○○님이 위치 공유를 일시 정지했어요" 표시

**가족 위치 조회**
1. [지도 탭] → 지도에 가족 구성원별 핀
2. 핀 탭 시 카드 표시: 닉네임, 마지막 갱신 시각, "전화하기/메시지" 버튼
3. 카드의 "길찾기" → 시스템 지도 앱으로 deeplink

#### F5.4 상세 규칙
- 갱신 주기: 적응적
  - 이동 중: 30초~1분
  - 정지 시: 5분 이상
  - 배터리 < 20%: 갱신 주기 자동 5분으로
- 정확도 등급: 도시 단위가 아닌 약 10~50m
- 위치 데이터 보관:
  - 무료: 24시간 이내만 (히스토리 미제공)
  - 프리미엄: 30일~1년
- 외부 노출: **절대 불가**. Firestore Security Rules로 가족 ID 일치 시에만 조회.
- 자녀의 위치 공유 일시 정지: 부모에게 푸시 알림 ("민서가 위치 공유를 1시간 정지했어요")
- 위치 갱신 실패가 30분 이상 지속: 가족에게 "○○님 위치를 받지 못하고 있어요" 표시

#### F5.5 빈 상태 / 에러
- 위치 권한 없음: "위치 권한을 켜야 가족 위치를 볼 수 있어요" + 설정 deeplink
- 가족이 위치 공유 안 함: 핀 회색, "공유 끔" 표시

### F6. 공유 캘린더

#### F6.1 개요
가족 전원이 보는 공유 캘린더. 단발/반복 일정, 일정별 알림, 학원 시간표 통합 표시.

#### F6.2 User Stories
- **US-6.1** parent로서, 가족 일정을 추가하고 모든 구성원에게 알릴 수 있다.
- **US-6.2** child로서, 본인 일정을 추가할 수 있다 (가족에게 표시).
- **US-6.3** 구성원으로서, 월/주/일 뷰로 일정을 볼 수 있다.
- **US-6.4** 구성원으로서, 일정 시작 전 알림을 받을 수 있다.
- **US-6.5** parent로서, 학원 시간표를 등록하면 캘린더에 자동 반복 표시된다.

#### F6.3 사용자 플로우

**일정 추가**
1. [캘린더 탭 → 우하단 + 버튼]
2. 종류 선택: 일반 일정 / 학원 / 생일·기념일
3. 입력 항목:
   - 제목 (필수)
   - 날짜·시간 (필수, 종일 토글)
   - 반복 (없음 / 매일 / 매주 / 매월 / 사용자 정의)
   - 참여자 (가족 구성원 멀티 선택, 기본: 전체)
   - 장소 (선택, 지도 핀)
   - 메모 (선택)
   - 알림 시각 (없음 / 시작 시 / 10분 전 / 1시간 전 / 1일 전 / 사용자 정의)
4. 저장 → 캘린더에 표시 + 그룹 채팅에 시스템 메시지 (선택적)

**일정 수정/삭제**
1. 일정 탭 → 상세 화면
2. 우상단 [수정] 또는 [삭제]
3. 반복 일정 수정 시: "이 일정만 / 이후 모두 / 전체" 선택

**월/주/일 뷰 전환**
- 상단 토글로 전환
- 월 뷰: 점/색깔로 일정 표시, 탭하면 하단 시트에 그날 일정
- 주 뷰: 시간대별 칸
- 일 뷰: 해당 일의 모든 일정 + 학원 시간표 + 위치 정보 통합

#### F6.4 상세 규칙
- 일정 제목: 1~100자
- 알림: 디바이스 기본 알림 + 인앱 푸시
- 반복 일정: RRULE (RFC 5545) 호환 형식 저장
- 참여자별 RSVP (Phase 2)
- 캘린더 색상: 가족 구성원별 색 자동 할당
- 외부 캘린더 연동: Phase 2 (Google Calendar import)

#### F6.5 빈 상태 / 에러
- 일정 0개: "오늘은 가족 일정이 없어요" + 일러스트
- 시간 충돌 경고: 동일 시간 다른 일정 있을 때 "다른 일정과 겹쳐요" 안내 (저장은 허용)

---

### F7. 학원 일정 / 시간표 통합

#### F7.1 개요
한국형 핵심 차별 기능. 학원명·요일·시간을 입력하면 캘린더에 반복 일정으로 자동 표기. Phase 2에서 Geofence + 픽업 알림 자동화.

#### F7.2 User Stories
- **US-7.1** parent로서, 학원을 등록하고 요일별 시간표를 입력할 수 있다.
- **US-7.2** parent로서, 학원 셔틀 시간·픽업 위치를 함께 등록할 수 있다.
- **US-7.3** 구성원으로서, 캘린더에서 학원 일정을 일반 일정과 구분하여 볼 수 있다.
- **US-7.4** parent로서, 방학·시험기간 등 특수 기간을 설정해 자동으로 일시 정지할 수 있다.

#### F7.3 사용자 플로우

**학원 등록**
1. [캘린더 탭 → + → 학원] 또는 [더보기 → 학원 관리]
2. 입력 항목:
   - 학원 이름 (필수)
   - 대상 자녀 (가족 내 child 멀티 선택)
   - 요일별 시간표:
     - [월] 16:00~18:00 (영어)
     - [수] 16:00~18:00 (영어)
     - [금] 17:00~19:00 (수학)
     - 자유 추가 가능
   - 학원 위치 (지도 핀, 선택)
   - 셔틀 정보 (선택): 출발지, 출발 시각, 도착지
   - 학원 색상 (자동 할당, 변경 가능)
3. 저장 → 캘린더에 반복 일정으로 표시
4. 캘린더에서 학원 일정은 학원 아이콘 (🏫) + 학원 색상으로 구분

**학원 수정/삭제**
- [더보기 → 학원 관리 → 해당 학원]에서 일괄 수정
- 시간표 부분 수정 시 변경 적용 시점 선택 ("오늘부터 / 다음 주부터")

**일시 정지 (방학 등)**
- 학원 상세에서 [기간 일시 정지]
- 시작·종료일 선택 → 해당 기간 캘린더에서 회색으로 표시 (또는 숨김 옵션)

#### F7.4 상세 규칙
- 학원당 시간표 슬롯: 무제한
- MVP에서 학원 수: 무료 5개, 프리미엄 무제한
- 학원 시간표는 캘린더 일반 일정과 같은 데이터 모델로 저장 (RRULE 활용)
- 학원 이벤트는 `eventType: "academy"`로 마킹, 필터링 가능
- 학원 알림: 시작 30분 전 부모에게 자동 알림 (사용자 변경 가능)
- (Phase 2) 셔틀 정보 + Geofence: 도착 시 자동 알림

#### F7.5 빈 상태 / 에러
- 학원 0개: "학원을 등록하면 캘린더에 시간표가 자동으로 표시돼요" + 등록 버튼
- 시간 충돌: 동일 자녀 동일 시간에 두 학원 등록 시 경고

---

### F8. 푸시 알림 / 알림 인박스

#### F8.1 개요
모든 알림(채팅, 일정, 위치, 학원, 시스템)을 통합 인박스로 관리. 종류별 ON/OFF 가능.

#### F8.2 User Stories
- **US-8.1** 구성원으로서, 새 메시지·일정·학원 알림을 푸시로 받을 수 있다.
- **US-8.2** 구성원으로서, 알림 종류별로 ON/OFF 할 수 있다.
- **US-8.3** 구성원으로서, 방해 금지 시간대를 설정할 수 있다.
- **US-8.4** 구성원으로서, 놓친 알림을 인박스에서 다시 볼 수 있다.

#### F8.3 알림 종류

| 알림 종류 | 트리거 | 기본 ON/OFF | 그룹 |
|---|---|:---:|---|
| 채팅 (그룹) | 새 메시지 | ON | 채팅 |
| 채팅 (1:1) | 새 메시지 | ON | 채팅 |
| 일정 임박 | 사용자 설정 시각 | ON | 일정 |
| 일정 추가됨 | 다른 구성원이 일정 추가 | ON | 일정 |
| 학원 시작 임박 | 30분 전 (변경 가능) | ON (parent) | 학원 |
| 학원 종료 임박 | 5분 전 (Phase 2) | OFF | 학원 |
| 위치 공유 변경 | 가족이 일시 정지/끄기 | ON (parent) | 위치 |
| 가족 가입/탈퇴 | 시스템 이벤트 | ON | 시스템 |
| 공지 | 운영자 공지 | ON | 시스템 |

#### F8.4 사용자 플로우

**알림 인박스**
1. [더보기 → 알림함] 또는 홈 우상단 종 아이콘
2. 종류별 탭(전체/채팅/일정/학원/위치/시스템)
3. 각 알림 탭 시 해당 화면으로 deeplink

**방해 금지**
- [더보기 → 설정 → 알림 → 방해 금지]
- 시작·종료 시각 (예: 22:00~07:00)
- 예외: 가족 SOS (Phase 2)

#### F8.5 상세 규칙
- 푸시 페이로드: title, body, deeplink, group
- 알림 그룹화: iOS Notification Grouping / Android Channel
- 무음 알림: 위치 갱신 등 데이터 전용 푸시는 사용자에게 미노출
- 인박스 보관: 30일

#### F8.6 빈 상태 / 에러
- 인박스 비어있음: "받은 알림이 없어요"
- 푸시 권한 거부: 인박스 상단 배너 "푸시 알림이 꺼져 있어요. 설정에서 켜기"

### F9. 프로필 / 설정

#### F9.1 개요
본인 프로필 관리, 가족 정보, 알림·위치·개인정보 설정, 약관 보기, 로그아웃·탈퇴.

#### F9.2 화면 구성

**내 프로필**
- 프로필 사진 (탭하여 변경)
- 닉네임 (탭하여 수정)
- 이메일/전화 (인증 정보 표시, 변경 가능)
- 역할 (parent/child, owner는 양도 시에만 변경)

**설정**
- 알림 (종류별 토글, 방해 금지)
- 위치 (공유 ON/OFF, 정확도, 데이터 보존 기간)
- 개인정보 (개인정보처리방침, 데이터 다운로드, 데이터 삭제 요청)
- 보안 (비밀번호 변경, 2단계 인증 — Phase 2)
- 디스플레이 (다크 모드, 언어, 글자 크기)
- 기타 (앱 버전, 라이선스, 문의)
- 로그아웃 / 탈퇴

#### F9.3 가족 관리
- 가족 이름·사진 변경 (parent+ 권한)
- 구성원 목록 (역할, 가입일, 마지막 활동)
- 초대하기 (F2)
- owner 양도 (현 owner만 가능)
- 가족 떠나기 (자기 자신)
- 가족 삭제 (owner만, 30일 유예 후 영구 삭제)

#### F9.4 데이터 권리 (PIPA 준수)
- **데이터 다운로드**: 내 메시지, 일정, 위치 히스토리, 사진을 ZIP/JSON으로 받기
- **데이터 삭제 요청**: 30일 내 처리, 비가역적 삭제
- **이용 정지 요청**: 탈퇴 없이 일시 정지

## 4. 화면 흐름 요약 (Critical Paths)

### 4.1 신규 사용자 → 가족 만들기 → 자녀 초대까지

```
[시작 화면]
  → 가입 (이메일/소셜)
  → 약관 동의
  → 프로필 설정
  → "가족 만들기" 선택
  → 가족 이름 입력
  → [홈] 진입 (빈 가족, 본인만)
  → [더보기 → 가족 관리 → 초대하기]
  → 링크 또는 QR 생성
  → 카톡으로 자녀에게 공유
  → 자녀가 링크 탭 → 앱 설치 → 가족 가입 흐름
```

### 4.2 자녀의 학원 종료 → 픽업 요청

```
[자녀 폰: 학원 종료]
  → [채팅 탭] 가족 채팅 진입
  → 입력창 [+] → "위치 공유" 선택
  → 1탭으로 "끝났어요. 데리러 와요" + 현재 위치 핀 첨부
  → 전송
  → 부모 폰 푸시 알림
  → 부모가 위치 보고 출발
```

### 4.3 학원 등록 → 캘린더 표시

```
[부모: 더보기 → 학원 관리 → +]
  → 학원명 "○○영어"
  → 대상 자녀 선택 (민서)
  → 시간표 [월 16:00~18:00], [수 16:00~18:00], [금 17:00~19:00]
  → 학원 위치 핀
  → 저장
  → [캘린더 탭]에 자동 반복 표시 (학원 아이콘 + 색상)
  → 시작 30분 전 부모 폰에 알림
```

- 모서리 반경 16dp (카드), 12dp (버튼)
