## 3. User Flow Diagrams

### 3.1 신규 사용자 → 가족 만들기 (Owner Flow)

```
[앱 첫 실행]
   │
   ▼
S01-SPLASH (1.5s 이내 분기)
   │ 미인증
   ▼
S02-SIGN-IN ──── "가입하기" ──→ S03-SIGN-UP
                                    │
                          (이메일/소셜/전화 선택)
                                    │
                                    ▼
                                S04-VERIFY (이메일/전화 인증)
                                    │
                                    ▼
                                S05-TERMS (필수/선택 약관)
                                    │
                                    ▼
                                S06-PROFILE (닉네임/사진/역할=parent)
                                    │
                                    ▼
                            S08-FAMILY-CHOICE
                              ├─ "가족 만들기" → S09-FAMILY-NEW
                              │                     │
                              │                     ▼
                              │                 S12-HOME (구성원=본인만)
                              │                     │
                              │                     ▼
                              │                 [가족 관리 → 초대]
                              │                     │
                              │                     ▼
                              │                 S10-INVITE (링크/QR 공유)
                              │
                              └─ "초대 코드/링크로 가입" → S11-INVITE-JOIN
```

### 3.2 신규 사용자 → 초대 링크로 가입 (Member Flow)

```
[가족이 보낸 카톡/문자의 초대 링크 클릭]

   ├─ 앱 미설치: 스토어로 이동 → 설치 → 앱 자동 실행 (deeplink 보존)

S01-SPLASH (deeplink 토큰 추출)
   │

   │                                            │
   │                                            ▼
   │                                       S12-HOME
   │
   └─ 인증됨: → S11-INVITE-JOIN 바로
```

### 3.3 만 14세 미만 자녀 가입 (Minor Flow)

```
S03-SIGN-UP (또는 S08에서 자녀로 분기)
   │
   ▼
S06-PROFILE에서 생년월일 입력 시 만 14세 미만 감지
   │
```

## 4. 화면별 상세 명세

### S01. 시작 화면 (Splash)

- **Screen ID**: S01-SPLASH
- **진입점**: 앱 콜드 스타트
- **목적**: 인증/세션 상태 확인 후 분기
- **분기 라우팅**:
  - 미인증 → S02-SIGN-IN
  - 인증됐고 가족 없음 → S08-FAMILY-CHOICE
  - 인증됐고 가족 있음 → S12-HOME
  - 딥링크 토큰 있음 + 인증됨 → S11-INVITE-JOIN
  - 딥링크 토큰 있음 + 미인증 → S03-SIGN-UP (토큰 보존)

**레이아웃**
```

│                                      │
│                                      │

```

**컴포넌트 명세**
- 배경: `--color-bg` (라이트) / `--color-primary-soft` (그라데이션도 가능)
- 로고: 100×100, 중앙
- 앱 이름: `display`, 로고 아래 24px
- 슬로건: `body`, `--color-text-secondary`
- 로딩 인디케이터: 24×24, primary 컬러

**상태**
| 상태 | 표시 |
|---|---|
| `loading` | 로고 + 로딩 인디케이터 |
| `error-network` | 토스트 "인터넷 연결을 확인하세요" + 재시도 버튼 |
| `error-auth` | 자동 로그아웃 처리 후 S02로 이동 |

**디자인 노트**
- 1.5초 이내 분기 처리. 그 이상이면 백그라운드에서 처리하되 화면은 다음으로 진행.
- iOS LaunchScreen.storyboard와 시각적으로 연속되도록.

### S02. 로그인 (Sign In)

- **Screen ID**: S02-SIGN-IN
- **진입점**: S01 미인증 / 명시적 로그아웃 후
- **목적**: 기존 계정 로그인 또는 가입 플로우 진입
- **다음 화면**: 성공 → S08(가족 없으면) / S12(가족 있으면), "가입하기" → S03

```
│                                      │
└──────────────────────────────────────┘
```

**컴포넌트 명세**
- 헤더: 로고 64×64, 슬로건 `heading-1`
- 이메일 입력: `Input.Text`, autocapitalize=off, keyboardType=email
- 비밀번호: `Input.Text`, secureText, 우측 보기/숨기기 토글
- 로그인 버튼: `Button.Primary`, 하단 24px gap
- 비밀번호 재설정: `Button.Ghost`, 텍스트 링크
- 소셜 버튼: 각각 `Button.Secondary` 변형, 좌측 아이콘 24×24
- 가입하기: 푸터에 `Button.Ghost`

**상태**
| 상태 | 동작 |
|---|---|
| `default` | 모든 필드 비어 있음, 로그인 버튼 disabled |
| `valid` | 이메일/비밀번호 입력 시 enabled |
| `loading` | 로그인 버튼 → 스피너, 입력 disabled |
| `error-credential` | 비밀번호 필드 아래 에러 텍스트 "이메일 또는 비밀번호가 일치하지 않아요" |
| `error-locked` | "여러 번 잘못 입력하셔서 5분 잠겼어요. 잠시 후 다시 시도해 주세요" |
| `error-network` | 토스트 "네트워크 연결을 확인하세요" |

**검증 규칙**
- 이메일: RFC 5322 기본 검증
- 비밀번호: 8자 이상 (서버에서 최종 검증)
- 5회 실패 후 5분 잠금 (서버 응답 기반)

**인터랙션**
- 키보드 enter → 다음 필드 / 마지막 필드 → 로그인 시도
- 비밀번호 보기/숨기기 토글
- 소셜 버튼 → OAuth 모달

**접근성**
- 입력 라벨 명확, screen reader용 `accessibilityLabel`
- 폰트 크기 200%까지 안전

## 5. 공통 컴포넌트 상세

### 5.1 하단 탭 바

```
┌──────────────────────────────────────┐
│  🏠     💬       🗺      📅      ⋯    │
│  홈    채팅    지도   캘린더   더보기 │
└──────────────────────────────────────┘
```

- 5개 탭, 높이 64 + Safe Area
- 활성 탭: 아이콘 채워짐 + `--color-primary`, 라벨 굵게
- 미확인 카운트 뱃지: 채팅·알림 옆 (홈 헤더에 알림 종)
- 탭 간 전환은 인스턴트 (애니메이션 100ms)

### 5.2 헤더 패턴

| 화면 | 헤더 |
|---|---|
| 메인 탭 | 좌측 페이지 타이틀, 우측 액션 (검색·+·종) |
| 상세 화면 | 좌측 ← 뒤로, 중앙 타이틀, 우측 액션 |
| 모달 | 좌측 × 닫기, 중앙 타이틀, 우측 [완료]/[저장] |

### 5.3 Empty State 컴포넌트

```
┌──────────────────────────────────────┐
│                                      │
```

## 6. 상태 패턴 (전역 표준)

| 상태 | 패턴 |
|---|---|
| `loading` | Skeleton 컴포넌트 (회색 박스 펄스 애니메이션) |
| `empty` | 5.3 Empty State 컴포넌트 |

| `offline` | 페이지 상단 노란 띠 "오프라인" |

## 7. 다크 모드 / 접근성 노트

### 7.1 다크 모드
- 모든 화면 다크 변형 필수
- 컬러 토큰만 변경 (구조는 동일)
- 사진/이미지: 자동 디머가 아닌 의도적 명도 보정 적용

### 7.2 폰트 크기 대응
- iOS Dynamic Type: XS ~ XXXL 모두 지원
- Android Font Scale: 0.85 ~ 2.0 지원
- 200% 확대 시 레이아웃 깨지지 않음 (가로 스크롤 허용 X, 세로 확장 OK)

### 7.3 색상 대비
- WCAG AA 기준: 본문 4.5:1, 큰 텍스트 3:1
- 가족 컬러는 본문 위에 직접 텍스트로 사용 금지 (도트·바·아바타 보더로만)

### 7.4 스크린 리더
- 모든 인터랙티브 요소에 `accessibilityLabel`
- 가족 컬러는 시각 정보뿐이므로, 스크린 리더에는 닉네임 명시
- 카드 그룹은 `accessibilityRole="group"`

### 7.5 햅틱 / 사운드
- 메시지 송신: 가벼운 햅틱
- 일정 추가 성공: 성공 햅틱
- 위치 정지 등 중요 변경: 중간 햅틱 + 시각 피드백
