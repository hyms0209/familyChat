# 부트스트랩 단계 A — poc-bank 분석 (파일 생성 없음)

| 항목 | 값 |
|---|---|
| 단계 | A / D |
| 예상 시간 | 1~3분 |
| 산출물 | 매핑 계획 표 4개 (채팅창에 마크다운으로 출력) |
| 파일 생성 | **절대 없음** (Read 만 사용) |
| 다음 단계 | `13_부트스트랩_B_프로젝트.md` |

> 이 단계는 **분석만** 한다. 파일·폴더를 만들지 않는다. poc-bank 핵심 파일 6개를 읽고 가족채팅에 어떻게 매핑할지 표로 출력한다.

---

## 사용 방법

Cursor의 Claude Code 채팅창에 **한 줄**만 입력하면 된다.

```
@13_부트스트랩_A_분석.md 의 [프롬프트] 섹션 그대로 실행해줘.
```

또는 Claude Code가 `@` 참조를 못 따라가면 아래 [프롬프트] 블록을 그대로 복사해서 붙여넣기.

---

## [프롬프트]

```
가족채팅 모바일 앱 부트스트랩 — 단계 A (분석만, 파일 생성 절대 금지)

▌ 엄격한 규칙
- Read 도구만 사용
- Write/Edit/bash 명령(mkdir, touch, npm, git 등) 일절 금지
- mobile/ 폴더 만들지 말 것
- 5분 안에 끝내야 함
- 결과는 마크다운 표로 채팅창에 출력만

▌ 읽을 파일 6개
1. /Users/lmh/Documents/work/source/react/busanpoc/poc-bank/package.json
2. /Users/lmh/Documents/work/source/react/busanpoc/poc-bank/tailwind.config.js
3. /Users/lmh/Documents/work/source/react/busanpoc/poc-bank/tailwind.plugin.font-styles.js
4. /Users/lmh/Documents/work/source/react/busanpoc/poc-bank/App.tsx
5. /Users/lmh/Documents/work/source/react/busanpoc/poc-bank/src/navigation/Router.ts
6. /Users/lmh/Documents/work/source/react/busanpoc/poc-bank/src/navigation/NavigationTypes.ts

▌ 출력할 4개 마크다운 표

A. 의존성 매핑
| 패키지 | poc-bank 버전 | 가족채팅에서 | 이유 |
- 가져올 패키지 (RN/Expo/공통 인프라)
- 제외할 패키지 (BNK·블록체인 등 도메인 종속)
- 추가할 패키지 (firebase, expo-location, expo-task-manager, expo-notifications, react-native-maps, @sentry/react-native)

B. Tailwind 토큰 매핑
| 카테고리 | poc-bank 값 | 가족채팅 값 |
- colors: poc-bank의 primary(blue), secondary, success/warning/error/info
  → 가족채팅의 primary(코랄 #FF7E6B), secondary(연한 블루 #7EB6E8) 등
  (가족채팅 컬러는 04_와이어프레임.md §1.2 표 참조)
- fontFamily: 둘 다 Pretendard 동일
- fontSize: poc-bank의 display-lg/headline-*/title-*/body-*/label-* 그대로 유지
- spacing: xs/sm/md/lg/xl/xxl/xxxl 그대로
- borderRadius: 그대로

C. Provider 매핑
| poc-bank Provider | 가족채팅에서 | 동일/대체/추가 |
- ModeProvider (User/Merchant) → FamilyProvider 로 대체 (가족 ID/구성원/역할)
- DeeplinkProvider, AuthCheckProvider, DialogProvider, ToastProvider, LoadingProvider, QueryProvider, DevToolsProvider → 동일 (Auth는 Firebase 기반으로 적응)

D. MENU 매핑
| poc-bank MENU | 가족채팅 MENU | 비고 |
poc-bank MENU 객체에서 가족채팅에 매핑 가능한 도메인 도출
가족채팅 도메인: SPLASH, AUTH, FAMILY, MAIN(HOME/CHAT/MAP/CALENDAR/MORE), CHAT(ROOM/ATTACHMENT), EVENT, ACADEMY, LOCATION, NOTIFICATIONS, SETTINGS

▌ 마지막 줄
"단계 A 완료 — 다음 단계 B (프로젝트 생성) 진행 가능" 으로 마쳐줘.
```

## 성공 기준

- [ ] 4개 마크다운 표가 채팅창에 모두 출력됨
- [ ] mobile/ 폴더 미생성 확인 (`ls /Users/lmh/Documents/Claude/Projects/가족채팅/mobile/ 2>/dev/null` 빈 출력)
- [ ] "단계 A 완료" 마지막 줄

## 실패 시 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| 5분 이상 응답 없음 | Cursor freeze | Cursor 강제 종료 후 재시작, 다시 시도 |
| poc-bank 폴더 접근 못 한다는 에러 | Add Folder to Workspace 안 됨 | Cursor → File → Add Folder to Workspace → poc-bank 추가 |
| 분석 도중 파일을 만들기 시작 | 프롬프트의 "파일 생성 금지" 무시 | 즉시 Stop → 채팅창에 "파일 생성하지 말고 분석 결과만 출력해" 재지시 |
| 표가 너무 길어 잘림 | Claude Code 응답 한도 | 채팅창에 "B 표만 다시 출력해" 식으로 부분 재요청 |
