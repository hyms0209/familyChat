## P-2.1. Button (4 변형 × 5 상태)

**[목적]**
앱 전반의 표준 버튼 컴포넌트를 모든 변형/상태로 정의.

**[참조 문서]**
- `04_와이어프레임.md` §1.7 (`Button.Primary`, `Button.Secondary`, `Button.Ghost`, `Button.Destructive`)

**[전제 조건]**
- 단계 1 완료
- 새 .pen 파일 `design/02_components.pen` 생성

**[프롬프트]**
```
[공통 시작 지시문 적용]

"Button" 컴포넌트를 다음 4 변형 × 5 상태 매트릭스로 만들어줘.

변형:
- Primary (배경 --color-primary, 텍스트 white)
- Secondary (배경 surface, 보더 1px primary, 텍스트 primary)
- Ghost (배경 transparent, 텍스트 primary, 호버 시만 배경)
- Destructive (배경 --color-error, 텍스트 white)

상태 (각 변형별로):
- default
- pressed (살짝 어두워지거나 눌림 효과)
- loading (스피너 + 텍스트 dim)
- disabled (opacity 0.4)
- focus (보더 외곽선 2px primary)

크기: 기본 높이 52, 모서리 12, 폰트 body-bold, 좌우 패딩 24
변형명을 각 행 좌측 라벨로, 상태명을 각 열 헤더로 배치 (4×5 그리드).

각 버튼 안의 라벨 텍스트는 "확인" (한국어)으로 통일.
```

**[성공 기준]**
- [ ] 4 변형 × 5 상태 = 20개 버튼이 그리드로 표시
- [ ] 컬러는 모두 토큰 사용
- [ ] Pencil의 Variants 기능으로 컴포넌트화 (재사용 가능)

## P-2.2. Input (Text / Search × 4 상태)

## P-2.4. Card (Basic / 일정 / 메시지 / 학원)

**[참조 문서]** `04_와이어프레임.md` §1.7, §4 S12-S20 (카드 사용 예)

**[프롬프트]**
```
[공통 시작 지시문 적용]

"Card" 컴포넌트를 4 변형으로 만들어줘. 모두 surface 배경, 그림자 sm, 모서리 16, 내부 패딩 lg(16).

1) Card.Basic: 헤더(heading-3) + 본문(body) + (선택) 우하단 액션 버튼
   샘플: "오늘 일정" / "16:00 영어학원" / [캘린더 보기]

2) Card.Event: 좌측에 4px 가족/학원 컬러 바 + 시작·종료 시간 + 제목 + 장소
   샘플: 그린 바, "16:00 - 18:00" / "민서 영어학원" / "서초동 ○○빌딩"

3) Card.Message (채팅 메시지 미리보기용): 좌측 Avatar.sm + (이름 + 메시지 + 시간) + 우측 미확인 카운트 뱃지
   샘플: 민서 아바타, "민서: 영어학원 끝났어요!" / "2분 전" / "3"

4) Card.Academy: 학원 색상 좌측 바 + 학원명 + 요일 시간표 리스트 + (선택) 액션
   샘플: 그린 바, "영어학원 (○○어학원)" / "월·수 16:00~18:00" + "금 17:00~19:00" + "[수정]"

각 카드는 default + pressed 상태 모두 표시.
일시 정지(회색 처리) 상태는 Card.Academy에 추가 변형으로.
```

- [ ] 일정/학원 카드의 시간 표기가 일관된 포맷

## 부록 A — 컴포넌트 라이브러리 정리

12개 컴포넌트가 모두 만들어진 후, 다음 정리 프롬프트를 한 번 실행 권장:

```
design/02_components.pen 캔버스에 만든 모든 컴포넌트를 다음 기준으로 정리해줘:

1) 캔버스 좌측 상단부터 카테고리별로 그룹화:
   - Form (Button, Input, Chip, ListItem)
   - Display (Avatar, Card, Divider, Badge)
   - Navigation (TabBar, Header)
   - Overlay (Modal, BottomSheet, Toast)
   - State (EmptyState)

2) 각 그룹 위에 카테고리 헤더 (heading-2)

3) 모든 컴포넌트는 Pencil의 Components(Variants) 기능으로 등록되어 있어야 함 — 이렇게 해야 화면 만들 때 끌어다 쓸 수 있음

4) 각 컴포넌트 옆에 사용처 메모 (예: Card.Event → 캘린더 화면, 홈 일정 카드)
```
